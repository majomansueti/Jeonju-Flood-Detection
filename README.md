# Flood Detection with Sentinel-1 SAR and Google Earth Engine

![Jeonju Flood Detection](images/Screenshot%202025-11-12%20at%209.42.14%20AM.png)

This repository contains an open-source pipeline for high-resolution flood detection using Sentinel-1 Synthetic Aperture Radar (SAR) imagery and the Google Earth Engine (GEE) platform. The methodology was validated against the severe flood event in Jeonju, South Korea (July 2023).

## Project Overview

Remote sensing via SAR is a powerful tool for disaster monitoring because microwave radar can penetrate cloud cover and operate at night. However, mid-sized municipalities often lack the technical and financial infrastructure for real-time flood mapping. 

By leveraging the free computing power of GEE and generating localized Antecedent Precipitation Index (API) thresholds, this pipeline aims to democratize flood risk detection. 

### Key Features
* **Sentinel-1 Change Detection:** Uses pre-event and post-event VV-polarized SAR imagery to detect inundated pixels via backscatter change ($\Delta$dB).
* **Multi-Layer Integration:** Combines detection outputs with API rain accumulation, SRTM elevation, slope data, and OpenStreetMap building footprints.
* **Cultural Heritage Risk Analysis:** Identifies vulnerable cultural properties using the generated flood boundaries.

## Methodology Snippet

The core detection mathematically relies on the change in sigma-nought ($\sigma^{\circ}$) backscatter:

```math
\Delta\text{dB} = \sigma^{\circ}_{\text{post}} - \sigma^{\circ}_{\text{pre}}
```

Water surfaces act as specular reflectors, dropping the radar backscatter significantly compared to dry land. A flood mask is generated by thresholding: $M(x,y) = 1 \text{ if } \Delta\text{dB}(x,y) < -1.5\text{dB}$.

## Validation (Jeonju, July 2023)
The pipeline correctly detected 4.23 km² of inundated land during the July 15-20 weather event, primarily concentrated along the Jeonjucheon river corridor. A threshold of $\Delta$dB = 1.5 dB was optimized to achieve an **F1-Score of 0.78**.
